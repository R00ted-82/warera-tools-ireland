⚔️ Battle Orders payload encryptor

Encrypts the Battle Orders module with a password using AES-GCM 256 (PBKDF2-SHA-256, 200k iterations). Runs entirely in your browser. Nothing is sent anywhere.

Use something long enough to resist offline brute-force. A passphrase of 4+ random words works well. Anyone with the page source can attempt to decrypt with any password they try.

What's in the blob, and how to use it
  • Output format: base64 of salt(16) || iv(12) || AES-GCM-256 ciphertext+tag. Single line, no quotes.
  • KDF: PBKDF2-SHA-256, 200,000 iterations. Salt and IV are random each time you encrypt.
  • To install: paste the output as the value of BO_ENCRYPTED_PAYLOAD in your index.html (piece 3 of the chat tells you where).
  • To rotate the password or update the BO code: re-run this tool with the new plaintext or password and replace the constant.
  • Decrypt verification happens via AES-GCM's authentication tag, so a wrong password always fails cleanly. No false unlocks.
